## Forgot Password

Request a password reset OTP. The OTP is sent to the user's email if the account exists.

```
POST /api/v1/auth/forgot-password
```

<Callout type="info" title="Silent Failure by Design">
  This endpoint always returns `{"sent": true}` regardless of whether the email exists. This prevents attackers from enumerating registered email addresses.
</Callout>

### Request Body

<TypeTable
  type={{
    email: {
      type: 'string',
      description: 'Email address to send the password reset OTP to.',
      required: true,
    },
  }}
/>

### Response

```json title="Response — 200 OK"
{
  "sent": true
}
```

### Example

<Tabs items={['cURL', 'JavaScript']}>
  <Tab value="cURL">
    ```bash
    curl -X POST http://localhost:3001/api/v1/auth/forgot-password \
      -H "Content-Type: application/json" \
      -d '{ "email": "jane@example.com" }'
    ```
  </Tab>
  <Tab value="JavaScript">
    ```js
    await fetch('/api/v1/auth/forgot-password', {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({ email: 'jane@example.com' }),
    });
    ```
  </Tab>
</Tabs>

## Reset Password

Reset a user's password using the OTP received via email. All existing sessions are revoked after a successful reset.

```
POST /api/v1/auth/reset-password
```

<Callout type="warn" title="Session Revocation">
  After a successful password reset, **all** of the user's active sessions are revoked. The user must sign in again with the new password.
</Callout>

### Request Body

<TypeTable
  type={{
    email: {
      type: 'string',
      description: 'Email address of the account being reset.',
      required: true,
    },
    otp: {
      type: 'string',
      description: 'The 6-digit OTP code received via email from the forgot-password request.',
      required: true,
    },
    new_password: {
      type: 'string',
      description: 'The new password. Must be at least 8 characters long.',
      required: true,
    },
  }}
/>

### Response

```json title="Response — 200 OK"
{
  "reset": true
}
```

### Password Reset Flow

<Steps>
  <Step>

    ### Request the OTP

    Call `POST /auth/forgot-password` with the user's email.

  </Step>
  <Step>

    ### Receive the OTP

    The user receives a 6-digit code via email.

  </Step>
  <Step>

    ### Submit the reset

    Call `POST /auth/reset-password` with the email, OTP, and new password.

  </Step>
  <Step>

    ### Sign in again

    All existing sessions are revoked. The user must sign in with the new password.

  </Step>
</Steps>

### Example

<Tabs items={['cURL', 'JavaScript']}>
  <Tab value="cURL">
    ```bash
    curl -X POST http://localhost:3001/api/v1/auth/reset-password \
      -H "Content-Type: application/json" \
      -d '{
        "email": "jane@example.com",
        "otp": "482937",
        "new_password": "newSecureP@ss456"
      }'
    ```
  </Tab>
  <Tab value="JavaScript">
    ```js
    await fetch('/api/v1/auth/reset-password', {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({
        email: 'jane@example.com',
        otp: '482937',
        new_password: 'newSecureP@ss456',
      }),
    });
    ```
  </Tab>
</Tabs>

### Errors

| Status | Code | Cause |
|---|---|---|
| `400` | `bad_request` | Invalid OTP, expired OTP, password too short |
| `401` | `unauthorized` | User not found or inactive |

## Change Password

Change the password for the currently authenticated user. Requires the current password for verification. All other sessions (except the current one) are revoked.

```
POST /api/v1/auth/change-password
```

<Callout type="info" title="Authentication Required">
  This endpoint requires a valid Bearer token. Only the current session is preserved — all other sessions are revoked.
</Callout>

### Request Body

<TypeTable
  type={{
    current_password: {
      type: 'string',
      description: 'The user\'s current password for verification.',
      required: true,
    },
    new_password: {
      type: 'string',
      description: 'The new password. Must be at least 8 characters long.',
      required: true,
    },
  }}
/>

### Response

```json title="Response — 200 OK"
{
  "changed": true
}
```

### Example

<Tabs items={['cURL', 'JavaScript']}>
  <Tab value="cURL">
    ```bash
    curl -X POST http://localhost:3001/api/v1/auth/change-password \
      -H "Authorization: Bearer YOUR_ACCESS_TOKEN" \
      -H "Content-Type: application/json" \
      -d '{
        "current_password": "mySecureP@ss123",
        "new_password": "evenMoreSecure789"
      }'
    ```
  </Tab>
  <Tab value="JavaScript">
    ```js
    await fetch('/api/v1/auth/change-password', {
      method: 'POST',
      headers: {
        Authorization: `Bearer ${token}`,
        'Content-Type': 'application/json',
      },
      body: JSON.stringify({
        current_password: 'mySecureP@ss123',
        new_password: 'evenMoreSecure789',
      }),
    });
    ```
  </Tab>
</Tabs>

### Errors

| Status | Code | Cause |
|---|---|---|
| `400` | `bad_request` | New password is too short (minimum 8 characters) |
| `401` | `unauthorized` | Current password is incorrect or invalid session |
