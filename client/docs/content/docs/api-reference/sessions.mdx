## Overview

Each time a user signs in (via credentials or OAuth), a new session is created. Sessions track device information, expiration, and activity timestamps. Users can view all their sessions and selectively revoke them.

<Callout type="info" title="Authentication Required">
  All session management endpoints require a valid Bearer token.
</Callout>

---

## List Sessions

Get all sessions for the currently authenticated user, ordered by creation date (newest first).

```
GET /api/v1/auth/sessions
```

### Response

Returns an array of session objects:

<TypeTable
  type={{
    id: {
      type: 'UUID',
      description: 'Unique session identifier. Use this ID to revoke a specific session.',
      required: true,
    },
    device_name: {
      type: 'string | null',
      description: 'Name of the device or browser. May be the User-Agent string if no explicit name was provided during sign in.',
    },
    expires_at: {
      type: 'DateTime<Utc>',
      description: 'When this session will expire.',
      required: true,
    },
    revoked_at: {
      type: 'DateTime<Utc> | null',
      description: 'When this session was revoked. Null if still active.',
    },
    last_seen_at: {
      type: 'DateTime<Utc> | null',
      description: 'Updated on each authenticated request. Shows when the session was last used.',
    },
    created_at: {
      type: 'DateTime<Utc>',
      description: 'When this session was created.',
      required: true,
    },
  }}
/>

### Example

<Tabs items={['cURL', 'JavaScript']}>
  <Tab value="cURL">
    ```bash
    curl http://localhost:3001/api/v1/auth/sessions \
      -H "Authorization: Bearer YOUR_ACCESS_TOKEN"
    ```
  </Tab>
  <Tab value="JavaScript">
    ```js
    const res = await fetch('/api/v1/auth/sessions', {
      headers: { Authorization: `Bearer ${token}` },
    });

    const sessions = await res.json();
    ```
  </Tab>
</Tabs>

## Revoke Session

Revoke a specific session by its ID. The session token becomes immediately invalid.

```
DELETE /api/v1/auth/sessions/{session_id}
```

### Path Parameters

<TypeTable
  type={{
    session_id: {
      type: 'UUID',
      description: 'The ID of the session to revoke. Must belong to the authenticated user.',
      required: true,
    },
  }}
/>

<Callout type="warn" title="Self-Revocation">
  You can revoke your own current session with this endpoint. The token used to make this request will become invalid immediately. Use `POST /auth/signout` if you want a cleaner signout experience.
</Callout>

### Response

```json title="Response — 200 OK"
{
  "revoked": true
}
```

### Example

<Tabs items={['cURL', 'JavaScript']}>
  <Tab value="cURL">
    ```bash
    curl -X DELETE http://localhost:3001/api/v1/auth/sessions/a1b2c3d4-e5f6-7890-abcd-ef1234567890 \
      -H "Authorization: Bearer YOUR_ACCESS_TOKEN"
    ```
  </Tab>
  <Tab value="JavaScript">
    ```js
    await fetch(`/api/v1/auth/sessions/${sessionId}`, {
      method: 'DELETE',
      headers: { Authorization: `Bearer ${token}` },
    });
    ```
  </Tab>
</Tabs>

### Errors

| Status | Code | Cause |
|---|---|---|
| `401` | `unauthorized` | Missing or invalid authentication token |

<Callout type="info" title="Idempotent">
  Revoking an already-revoked session is a no-op — the endpoint returns `{"revoked": true}` without error. The session must belong to the authenticated user; sessions belonging to other users are silently ignored.
</Callout>
