## Base URL

All API endpoints are served under the `/api/v1` prefix. The server also exposes a health check at the root level.

```
GET /health → "ok"
GET /api/v1/auth/signup
GET /api/v1/auth/signin
...
```

## Rate Limiting

The API applies a global rate limit using [Tower Governor](https://github.com/benwis/tower-governor):

- **Rate:** 1 request per second (sustained)
- **Burst:** Up to 120 requests

<Callout type="warn" title="Rate Limited">
  When you exceed the rate limit, the server returns a `429 Too Many Requests` response. Back off and retry with exponential delay.
</Callout>

## Quick Start

<Steps>
  <Step>

    ### Set up your environment

    Clone the repository and configure environment variables:

    ```bash title=".env"
    POSTGRES_DB=forkfit
    POSTGRES_USER=your_user
    POSTGRES_PASSWORD=your_password
    REDIS_HOST=localhost
    REDIS_PORT=6379
    GOOGLE_CLIENT_ID=your-google-client-id
    GOOGLE_CLIENT_SECRET=your-google-client-secret
    GITHUB_CLIENT_ID=your-github-client-id
    GITHUB_CLIENT_SECRET=your-github-client-secret
    ```

  </Step>
  <Step>

    ### Start the server

    ```bash
    cargo run
    ```

    The server will automatically run database migrations and seed default roles/permissions on startup.

  </Step>
  <Step>

    ### Verify the server is running

    ```bash
    curl http://localhost:3001/health
    # → ok
    ```

  </Step>
  <Step>

    ### Create your first account

    <Tabs items={['cURL', 'JavaScript']}>
      <Tab value="cURL">
        ```bash
        curl -X POST http://localhost:3001/api/v1/auth/signup \
          -H "Content-Type: application/json" \
          -d '{
            "email": "user@example.com",
            "password": "securepassword123",
            "full_name": "John Doe"
          }'
        ```
      </Tab>
      <Tab value="JavaScript">
        ```js
        const response = await fetch('http://localhost:3001/api/v1/auth/signup', {
          method: 'POST',
          headers: { 'Content-Type': 'application/json' },
          body: JSON.stringify({
            email: 'user@example.com',
            password: 'securepassword123',
            full_name: 'John Doe',
          }),
        });

        const data = await response.json();
        console.log(data.access_token);
        ```
      </Tab>
    </Tabs>

  </Step>
</Steps>

## Database Seeding

On startup the server automatically:

1. Creates an **admin** role with description "Full platform administrator"
2. Creates a **user** role with description "Default application user"
3. Seeds default permissions for the admin role:
   - `roles:read`, `roles:create`
   - `role_permissions:manage`
   - `permissions:read`, `permissions:create`
   - `user_roles:manage`
4. If `ADMIN_EMAIL` is set and that user exists, assigns the admin role to them
