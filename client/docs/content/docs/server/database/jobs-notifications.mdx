# Background Jobs & Notifications

This module schedules asynchronous tasks (such as overnight meal plan generations) and logs transactional notifications.

## 1. Background Jobs Queue

### `background_jobs`
Defines a transactional database-backed worker queue supporting retries and scheduled execution.

* **Schema**:
  | Column | Type | Constraints | Default | Description |
  | :--- | :--- | :--- | :--- | :--- |
  | `id` | `uuid` | `PRIMARY KEY` | `gen_random_uuid()` | Unique job identifier. |
  | `queue_name` | `text` | `NOT NULL` | `'default'` | Worker lane targeting (e.g. `'default'`, `'high_priority'`). |
  | `job_type` | `text` | `NOT NULL` | - | Action name (e.g., `'meal_plan_generation'`). |
  | `payload` | `jsonb` | `NOT NULL` | `'{}'` | Task parameters passed to the background handler. |
  | `status` | `text` | `NOT NULL` | `'queued'` | State: `'queued'`, `'processing'`, `'completed'`, or `'failed'`. |
  | `attempts` | `integer` | `NOT NULL` | `0` | Number of executed attempts. |
  | `max_attempts` | `integer` | `NOT NULL` | `3` | Maximum retry limit before marking as failed. |
  | `run_at` | `timestamptz` | `NOT NULL` | `now()` | Target scheduled execution time. |
  | `locked_at` | `timestamptz` | - | - | Lock timestamp when a worker claims the task. |
  | `locked_by` | `uuid` | - | - | Worker process identity uuid. |
  | `started_at` | `timestamptz` | - | - | Execution start timestamp. |
  | `completed_at` | `timestamptz` | - | - | Execution complete timestamp. |
  | `error_log` | `text` | - | - | Error stack trace if the execution failed. |
  | `created_at` | `timestamptz` | `NOT NULL` | `now()` | Record creation timestamp. |
  | `updated_at` | `timestamptz` | `NOT NULL` | `now()` | Record change timestamp. |

## 2. Notification Audit Log

### `notification_logs`
Chronicles all outgoing transactional customer alerts.

* **Schema**:
  | Column | Type | Constraints | Default | Description |
  | :--- | :--- | :--- | :--- | :--- |
  | `id` | `uuid` | `PRIMARY KEY` | `gen_random_uuid()` | Audit log identifier. |
  | `user_id` | `uuid` | `REFERENCES users(id) ON DELETE SET NULL` | - | Target recipient user profile. |
  | `channel` | `text` | `NOT NULL` | - | Target delivery mechanism: `'email'` or `'sms'`. |
  | `recipient` | `text` | `NOT NULL` | - | Target destination (e.g., phone number or email address). |
  | `status` | `text` | `NOT NULL` | `'queued'` | Delivery status: `'queued'`, `'sent'`, or `'failed'`. |
  | `provider` | `text` | - | - | Sending provider (e.g., `'sendgrid'`, `'twilio'`). |
  | `provider_message_id` | `text` | - | - | Unique tracking ID generated by provider. |
  | `error_message` | `text` | - | - | Error warning message returned if sending failed. |
  | `sent_at` | `timestamptz` | - | - | Timestamp when provider confirmed dispatch. |
  | `created_at` | `timestamptz` | `NOT NULL` | `now()` | Record creation timestamp. |
