# Signup & Signin 

All authentication routes are nested under `/api/v1/auth`.

---

## 1. Authentication Handlers

### Sign Up Account

`POST /api/v1/auth/signup`

Registers a new user account on the platform.

* **Payload Parameters**:
  <TypeTable
    type={{
      email: { type: 'string', required: true, description: 'Unique user email address.' },
      password: { type: 'string', required: true, description: 'Min 8 characters password.' },
      full_name: { type: 'string', required: true, description: 'User full name.' }
    }}
  />

---

### Sign In Session

`POST /api/v1/auth/signin`

Logs a user in and returns a bearer access token.

* **Payload Parameters**:
  <TypeTable
    type={{
      email: { type: 'string', required: true, description: 'Login email.' },
      password: { type: 'string', required: true, description: 'Account password.' }
    }}
  />

---

## 2. Session Controls

### List Active Sessions

`GET /api/v1/auth/sessions`

Returns all active device login sessions for the authenticated user.

---

### Revoke Specific Session

`DELETE /api/v1/auth/sessions/{id}`

Terminates a specific session, instantly logging that device out.

* **Path Parameters**:
  <TypeTable
    type={{
      id: { type: 'string', required: true, description: 'The UUID of the session to revoke.' }
    }}
  />
