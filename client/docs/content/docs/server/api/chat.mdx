# Chat & Feedback 

Endpoints managing communication threads with the AI assistant and logging user reviews. All chat routes are nested under `/api/v1/threads` and `/api/v1/feedback`.

---

## 1. Chat Threads

### List Chat Threads

`GET /api/v1/threads`

Retrieves a list of conversational threads generated for the user.

---

### Create Chat Thread

`POST /api/v1/threads`

Initializes a new conversational session.

* **Payload Parameters**:
  <TypeTable
    type={{
      title: { type: 'string', description: 'Optional custom topic header for the thread.' }
    }}
  />

---

### Get Thread Details

`GET /api/v1/threads/{id}`

Retrieves details for a specific conversational thread ID.

---

### Delete Chat Thread

`DELETE /api/v1/threads/{id}`

Removes a chat thread and all its stored message logs.

---

### List Messages in Thread

`GET /api/v1/threads/{id}/messages`

Returns a list of conversation logs for a specific thread ID.

---

### Post Message to Thread

`POST /api/v1/threads/{id}/messages`

Appends a new user message to the conversation, which triggers background agent processing.

---

## 2. Platform Feedback

### Submit User Feedback

`POST /api/v1/feedback`

Submits user ratings and comments regarding system performance or AI advice accuracy. This feedback is processed asynchronously by the reflection engine to extract long-term user preferences.
